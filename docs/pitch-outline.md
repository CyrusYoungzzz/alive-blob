## 模块 2 — 做了什么 & 给谁做的（90s）

**画面**：办公场景 → 情绪链条图 → Blob 特写 → 视频 Demo

### 给谁做的？

> "你们有没有遇到过这种情况 — 老板莫名其妙冲你发了一顿脾气，你什么都没做错。"
>
> "你憋着，回到工位，找同事吐槽。同事听了也难受。晚上回家，你又不小心把情绪带给了家人。"
>
> "这就是**情绪垃圾桶链条** — 老板倒给你，你倒给朋友，朋友���给家人。每个人都在当别人的情绪垃圾桶，每个人都在被伤害。"

**目标用户**：办公族 — 工作压力大、和上级沟通不顺畅、有情绪但没有安全出口的人。

### 做了什么？

> "我们想**终结这条链条**。让情绪有一个出口，但这���出口不是另一个人。"
>
> "让 AI 来当你的情绪垃圾桶 — 不会受伤，不会记仇，不会把你的负能量再传给下一个人。"

> "这就是 Alive Blob — 一个你可以随便揉捏、怎么打都不会坏、还会用表情回应你的软体小生物。把讨厌的人的脸贴上去，打它，它替他挨。**链条到这里断掉。**"

### 视频 Demo（30-45s）

按顺序展示：
1. 全景：桌上的 Blob，圆屏亮着表情（3s）
2. 手指触摸 → 表情立刻变化 + 粒子特效（5s）
3. 触摸不同部位 → 不同反应（Curious / Shy）（5s）
4. 气动双腿行走特写 — 硅胶腿一伸一缩（8s）
5. 不同情绪 = 不同步态（平稳走 vs 暴躁跺脚 vs 犯困暂停）（8s）
6. 手机控制面板操作（5s）
7. AIGC 换脸：上传照片 → 生成 7 种表情视频 → Blob 顶着那张脸走路（10s）

### 视频后口头补充（15s）

> "你刚才看到的一切 — 触摸感应、7 种情绪、气动行走、AI 换脸 — 全部运行在一个树莓派上，BOM 不到 550 块。"

### 为什么是软体？

> "因为硬邦邦的机器人捏着不舒服，也不好笑。软体天生有**生命感**和**滑稽感** — 一个软乎乎的东西笨拙地走路，本身就让人想笑。"

## 模块 3 — 48 小时的故事（60s）

**画面**：决策树/分叉图 → 关键调试照片 → 完成度总结

**核心叙事线**：不是流水账，而是「每一步都在做选择题，每道题都踩了坑」。

### 选择 1：为什么做人形，不做抽象形态？

> "一开始我们也想过做一个抽象的软体 — 一坨会动的东西就行。但我们很快意识到：**你要打它出气，它得像个人。**"
>
> "抽象形态没有代入感。你不会对一个球发火，但你会对一张脸发火。人形 = 代入感 = 出气闭环成立。"

### 选择 2：为什么用气泵，不用电机/传动结构？

> "舵机和齿轮做出来的东西是硬的 — 你捏不动它，它也不好笑。"
>
> "我们需要的是**软的、能被揉捏的、还能自己动**的东西。气泵 + 硅胶是唯一同时满足这三个条件的方案。"

### 踩坑：气泵与硅胶的调试

> "但气动不是拿来就能用的。"

- **硅胶硬度**：太软撑��起来，太硬充气不动 — 反复试了几种配比才找到平衡点
- **阀门选择**：一开始用的阀门泄气太慢，腿缩不回来，走路变成原地抽搐 — 换了电磁阀才解决
- **可靠性**：气泵连续运行会过热 — 加了 30 秒看门狗 + 启动前强制泄压

> "48 小时里，光调硅胶腿的充放气节奏就花了大半天。但也正是因为踩了这些坑，我们才真正理解了软体行走的 know-how。"

### 完成度强调

> "这不是一个 PPT 项目。48 小时内，我们交付了：
> - 完整硬件组装 + 3 个软件模块
> - 3种情绪 × 3 种步态的完整映射
> - 手机实时控制 + AIGC 换脸
> - 三层安全机制确保硬件不会炸
>
> 它**现在就能走、能摸、能换脸。**"

## 模块 4 — 技术·壁垒·差异性（80s）

**画面**：三个创新点依次展开，每个配对比图/动图

**只讲三个点，每个 ~25s。重点不是参数，是「为什么只有我们能做」：**

### 创新 1：气动 = 唯一用"肌肉"走路的桌面机器人

> "商业产品中零竞品。"

> "传统桌面机器人用电机和齿轮 — 精确，但像机器。我们用气泵驱动硅胶腿，它会肉眼可见地呼吸、充气、收缩 — 不像电机驱动那样精确机械，而像生物一样有生命感。"

- 一个气泵 + 两个电磁阀 = 双腿交替膨胀收缩
- 零齿轮、零轴承 — 没有磨损件，天然耐揉捏
- 你捏它的时候手感是软的、温的、会回弹的 — 电机机器人做不到

**壁垒**：气动软体行走的 know-how（硅胶硬度、充放气节奏、阀门选型）全是我们踩坑踩出来的，没有现成方案可抄。

### 创新 2：打它一下，<100ms 全身响应

> "从手指按下，到表情变化 + 步态切换，不到 100 毫秒。"

- 12 通道电容触摸 → 区分头部/身体/腿部
- 触摸模式识别（单击/连击/长按/多点触控）→ 情绪状态机 → 同时驱动表情 + 步态
- 不是「按按钮播动画」— 是触觉、视觉、运动三条线实时联动的状态机

**壁垒**：多模态实时联动，延迟低到你感觉不到 — 这是「活的」和「玩具」的分水岭。

### 创新 3：把讨厌的人的脸贴上去

> "上传一张照片，AI 自动生成 7 种表情动画。现在这个 Blob 顶着他的脸，在你桌上被你随便捏。"

- 你打它，它用**那张脸**露出委屈/害怕的表情 — 出气闭环达成
- 角色热插拔：换一张照片，它就变成另一个人
- 本质：把抽象的情绪压力，绑定到一个**具体的、可捏的、无害的**物理对象上

**壁垒**：不是套壳 API — 是人脸检测 + 多情绪批量生成 + 角色热插拔的完整 pipeline。

### ���异性总结

> "市面上的桌面机器人，要么是硬壳轮式的（Cozmo/Vector），要么是纯屏幕虚拟宠物。没有一个是**软的、会用肌肉走路、能把人脸贴上去打**的。这个组合，只有我们有。"

**画面**：市场定位矩阵 → 三步走路径 → 愿景画面

|  | 低价 | 高价 |
|--|------|------|
| **情感交互** | **Alive Blob ← 我们在这** | Sony Aibo / Lovot |
| **工具型** | 扫地机器人 | 工业协作臂 |

> "情感交互机器人目前只有两个极端 — 要么是几万块的 Lovot，要么是手机上的虚拟宠物。中间有一个巨大的空白。"

### 商业路径（三步走）

1. **开源社区（现在）** — 硬件 BOM + 软件全开源，吸引 Maker 社区，建立品牌认知
2. **教育套件（6 个月）** — STEM 教育市场，一个套件覆盖气动原理 + 编程 + AI 三个学科
3. **消费级产品（12 个月）** — 桌面陪伴宠物，¥299-599 定价区间

### 数字支撑

- BOM ¥350-550 → 量产成本可压到 ¥150 以下
- 全球 STEM 教具市场 ~$3.6B（2025），年增长 15%+
- 伴侣机器人市场 ~$5.5B（2025），情感需求是最强驱动力

### 更大愿景

> "Alive Blob 不只是一个解压玩具。我们想证明：**让机器人活起来，不需要百万预算和博士团队。**"
>
> "未来每个人的桌上、床头、工位旁，都可以有一个懂你情绪、会用身体语言回应你的小生物。我们想做**情感机器人的宜家** — 人人买得起，人人能定制。"

## 附：Q&A 预案

| 可能的问题 | 建议回答 |
|-----------|---------|
| 为什么用气动不用电机？ | 软体 = 柔软触感 + 无磨损 + 成本低；电机做不到「捏着舒服」这件事 |
| 和 Vector/Cozmo 的区别？ | 它们是刚性轮式机器人；我们是软体步行 + 触感 + 可换脸，物理交互维度不同 |
| 安全性怎么保证？ | 三层看门狗（泵时间/泄压/GPIO 归零），即使软件崩溃硬件也安全 |
| 商业化最大挑战？ | 量产模具和供应链；但核心器件都是标准工业品，无定制 ASIC |
| AI 部分具体做了什么？ | AIGC 表情生成（即梦 API），未来计划加入语音��互和情绪学习 |
| 能量续航？ | 目前外接 12V 适配器；消费版计划用 3S 锂电池，预估 2-3 小时续航 |
| 为什么是解压不是陪伴？ | 解压是切入点，陪伴是终局 — 先用最强痛点打穿市场，再拓展场景 |

## 演讲技巧提醒

1. **视频 Demo 是核心武器** — 让评委「看到」而不是「听到」，视频播放时少说话
2. **技术深度选择性展开** — 6 分钟也不要贪多，只讲气动 + 全链路延迟 + AIGC 三个点
3. **数字要脱口而出** — ¥350、<100ms、48 小时、7 种情绪、4 种步态
4. **48 小时故事要有细节** — 一个翻车时刻 + 一个高光时刻，比列时间线更打动人
5. **结尾要有记忆点** — 最后一句话是评委记住你的关键
6. **准备一个 30 秒版本** — 万一被要求缩短，只讲：软体行走 + 触摸表情 + ¥350 + 48 小时
