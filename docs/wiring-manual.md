# Alive Blob - 接线操作手册

写给零基础的人。每一步都告诉你接什么、怎么接、为什么这么接。

## 开始之前

### 你需要准备的工具

- **十字螺丝刀（小号）**：拧继电器模块和降压模块上的接线端子螺丝
- **万用表**（或任何能测电压的工具）：调降压模块输出电压用
- **剥线钳或小刀**：剥线头的绝缘皮
- **剪刀**：剪气管

### 认识你的零件

开始接线之前，先把���有零件摊在桌上，认一遍：

| 零件 | 长什么样 | 干什么用的 |
|------|---------|----------|
| **Raspberry Pi 4B** | 信用卡大小的绿色电路板，有 USB、HDMI、40 针排针 | 整个系统的大脑，运行所有软件 |
| **2.1 寸圆屏 + 驱动板** | 一个小圆屏 + 一块长条形驱动板，有 HDMI 口和 USB 口 | Blob 的「脸」，显示表情视频 |
| **micro-HDMI 转 HDMI 线** | 一头小（micro-HDMI）一头大（标准 HDMI）的视频线 | 把 Pi 的视频信号传给圆屏 |
| **4 路继电器模块** | 红色/蓝色 PCB 板，上面有 4 个蓝色方块（继电器）| 用 Pi 的微弱信号控制 12V 大功率设备（气泵、阀门）|
| **LM2596S 降压模块** | 一小块 PCB，上面有个黑色电感线圈和螺丝端子 | 把 12V 电压降到 5V，给 Pi 和圆屏供电 |
| **12V 电源适配器** | 墙插电源，输出头是圆形 DC 5.5mm | 整个系统的电源，提供 12V |
| **12V 气泵** | 一个小马达模样的东西，有两根线和一个气管口 | 往硅胶腿里充气，驱动走路 |
| **12V 电磁阀 x2** | 小���柱体，两根线 + 两个气管口 | 控制气体通向哪条腿。通电 = 打开，断电 = 关闭 |
| **MPR121 触摸传感器** | 小 PCB 板，标有 SDA、SCL、VCC、GND | 检测用户的触摸，告诉 Pi 被摸了哪里 |
| **USB-C 供电线（公头带线）** | USB-C 接头 + 红黑两根裸线 | 把降压模块的 5V 输出接到 Pi 的 USB-C 供电口 |
| **杜邦线** | 彩色细线，两头有塑料插头 | 连接各个模块之间的信号和电源 |
| **面包板** | 白色塑料板，满是小孔 | 不用焊接就能连接电线的工具，用来做 GND 共地总线 |
| **6mm 硅胶气管** | 透明软管 | 连接气泵、电磁阀和硅胶腿 |

### 认识杜邦线

���邦线两头的插头有两种：

- **公头（pin）**：露出一根金属针，能插进面包板的孔里
- **母头（socket）**：是个小孔，能套在 Pi 的排针或模块的针脚上

你会用到：
- **公对母**：一头插面包板，一头套模块针脚
- **母对母**：两头都套针脚，用于模块与模块直连

### 认识面包板

面包板是一块白色塑料板，上面密密麻麻全是小孔。规则很简单：

```
面包板内部连接规则：

  + 电源轨 ──────────────── 整行横���连通
  - 电源轨 ──────────────── 整行横向连通

  a b c d e │ f g h i j
  ● ● ● ● ● │ ● ● ● ● ●    ← 同一行的 a-e 五个孔互通
            │                  中间有条沟，左右不通
            │                  同一行的 f-j 五个孔互通
```

**关键概念：** 把两根线插到面包板同一行（同侧），它们就连上了。不需要焊接。

**我们的用法：** 面包板主要用来做 **GND（接地）总线** — 很多根 GND 线都要连在一起，全部插到面包板的 `-` 轨就行了。

### 认识 Pi 的 GPIO 针脚

Pi 4B 有 40 根针脚（2 行 x 20 列），编号方式如下。**把 Pi 的 USB 口朝向自己**，针脚在你对面那边：

```
                    （Pi 板子对面那一边）
      ┌──────────────────────────────────────────┐
      │  3V3 (1) (2)  5V                         │
      │  SDA (3) (4)  5V                         │
      │  SCL (5) (6)  GND                        │
      │   4  (7) (8)  14                         │
      │  GND (9) (10) 15                         │
      │   17 (11)(12) 18  ← GPIO18 = 气泵控制     │
      │   27 (13)(14) GND                        │
      │   22 (15)(16) 23  ← GPIO23 = 左腿阀门     │
      │  3V3 (17)(18) 24  ← GPIO24 = 右腿阀门     │
      │   10 (19)(20) GND                        │
      │    9 (21)(22) 25                         │
      │   11 (23)(24)  8                         │
      │  GND (25)(26)  7                         │
      │    0 (27)(28)  1                         │
      │    5 (29)(30) GND                        │
      │    6 (31)(32) 12                         │
      │   13 (33)(34) GND                        │
      │   19 (35)(36) 16                         │
      │   26 (37)(38) 20                         │
      │  GND (39)(40) 21                         │
      └──────────────────────────────────────────┘
                （USB 口在这边，朝向你）
```

**我们用到的针脚：**

| 物理针脚号 | 功能 | 接到哪里 |
|-----------|------|---------|
| 3 | GPIO2 / SDA | MPR121 触摸传感器 SDA |
| 5 | GPIO3 / SCL | MPR121 触摸传感器 SCL |
| 6 | GND | 面包板 GND 总线 |
| 12 | GPIO18 | 继电器 IN1（气泵）|
| 16 | GPIO23 | 继电器 IN2（左腿阀门）|
| 17 | 3.3V | MPR121 VCC |
| 18 | GPIO24 | 继电器 IN3（右腿阀门）|

## 接线步骤

**总原则：先接低压 (5V)，再接高压 (12V)。全程不通电，全部接好再上电。**

---

### 第一步：搭建 GND 总线（面包板）

**做什么：** 在面包板上建立一条公共接地线，所有模块的 GND 都接到这里。

**为什么：** 电路里所有设备必须共用同一个「地」（GND），电流才能形成回路。就像水管需要有回水管一样。

**操作：**

1. 把面包板放在桌上
2. 找到面包板边上标着 **`-`** 的那一行（通常是蓝色线旁边的一排孔），这就是我们的 GND 总线
3. 后面每次说「接到 GND 总线」，就是把线的公头插到这一行的任意一个孔里

暂时不接任何线，面包板摆好就行，后面每步都会往这里接 GND。

---

### 第二步：降压模块接线（12V → 5V）

**做什么：** 把 12V 电源适配器的电连到降压模块，降压模块输出 5V。

**为什么：** Pi 和圆屏需要 5V 供电，但我们的电源是 12V 的。降压模块就是个电压转换器。

**认识降压模块的接口：**

```
降压模块上有 4 个螺丝端子：

  IN+  IN-  │  OUT+  OUT-
  (输入侧)   │  (输出侧)
  接 12V     │  输出 5V
```

**操作：**

1. **暂时不接 12V 电源适配器**（最后统一上电）
2. 准备两根杜邦线（或直接用电源适配器的线），先记住：
   - 12V 适配器的 **+（正极）** → 降压模块 **IN+**
   - 12V 适配器的 **-（负极）** → 降压模块 **IN-**
3. 从降压模块 **IN-** 引一根线到 **面包板 GND 总线** — 这样 12V 的地和 5V 的地就连通了

> **你可能在想：12V 电源适配器是个圆头插头，怎么接到螺丝端子上？**
>
> 买一个 **DC 母座接线端子**（几毛钱），圆插头插进母座，母座背面有螺丝端子可以引出正负极线。或者直接剪掉圆头，剥出里面两根线（中间的是 +，外面包着的是 -）。

---

### 第三步：调节降压模块输出电压

**做什么：** 在接 Pi 之前，先让降压模块输出准确的 5V。

**为什么：** Pi 4B 对供电电压敏感。低于 4.8V 会报欠压警告，高于 5.25V 可能烧毁。必须先调好再接。

> **注意：你买的是固定 5V 输出的 LM2596S，理论上出厂就是 5V，不需要调。但仍然建议上电后用万用表确认一下输出确实是 5V 左右（4.9V-5.2V 都可以）。如果电压不对，说明买错型号了（买成了可调版），需要用螺丝刀调节板上的电位器。**

**操作：**

1. 把 12V 电源适配器接到降压模块的 IN+/IN-（此时只有降压模块通电，其他什么都没接）
2. 插上电源适配器，通电
3. 用万用表的直流电压档，红表笔碰 **OUT+**，黑表笔碰 **OUT-**
4. 读数应该在 **4.9V ~ 5.2V** 之间
5. 确认没问题后，**拔掉电源适配器**，继续下面的步骤

---

### 第四步：Pi 供电线（降压模块 → USB-C → Pi）

**做什么：** 把降压模块的 5V 输出通过 USB-C 线接到 Pi 的供电口。

**为什么：** Pi 4B 用 USB-C 口供电。我们不用充电器，而是用降压模块提供 5V。

**操作：**

1. 拿出 **USB-C 公头供电线**（红黑两根裸线 + USB-C 头）
2. **红线** → 降压模块 **OUT+** 螺丝端子（用螺丝刀拧松端子螺丝，把裸线头插进去，拧紧）
3. **黑线** → 降压模块 **OUT-** 螺丝端子（同样操作）
4. USB-C 头先不插 Pi，等全部接好再插

```
降压模块 OUT+  ←── 红线 ──→  USB-C 公头 ──→ (待插入 Pi USB-C)
降压模块 OUT-  ←── 黑线 ──┘
```

---

### 第五步：圆屏供电和视频

**做什么：** 给圆屏驱动板供电 (5V USB) + 接视频信号 (HDMI)。

**为什么：** 圆屏需要两根线：一根传视频画面（HDMI），一根供电（USB 5V）。

**操作：**

1. **视频线：** 拿 micro-HDMI 转 HDMI 线：
   - **小头（micro-HDMI）** → 插 Pi 4B 的 micro-HDMI 口（Pi 有两个，用**靠近 USB-C 供电口的那个**，标记为 HDMI0）
   - **大头（标准 HDMI）** → 插圆屏驱动板的 HDMI 口

2. **供电线：** 圆屏驱动板有一个 **micro-USB 或 USB 供电口**，需要 5V：
   - 方案 A：用一根旧 USB 线，一头剪掉剥出红（+）黑（-）线，红接降压模块 OUT+，黑接 OUT-
   - 方案 B：从降压模块 OUT+ 和 OUT- 引两根杜邦线，接到面包板同一行，再从面包板引线接到驱动板的 5V/GND 接口（如果驱动板有裸针脚的话）

> **提示：** 最简单的办法是找一根旧的 USB 线给圆屏驱动板供电，USB 一头插驱动板，另一头剪掉接降压模块。

---

### 第六步：继电器模块接线 — 信号侧

**做什么：** 给继电器模块供电 (5V)，并把 Pi 的 GPIO 信号线接到继电器的输入端。

**为什么：** Pi 的 GPIO 只能输出 3.3V / 几毫安的微弱信号，直接带不动 12V 的气泵和阀门。继电器相当于一个「电子开关」—— Pi 给一个小信号，继电器就帮你接通/断开 12V 大电路。

**认识继电器模块的接口：**

```
继电器模块俯视图：

  信号输入侧（排针）：
  ┌─────────────────────────┐
  │ VCC  IN1  IN2  IN3  IN4 │   ← 从上往下或从左往右
  │ GND                     │
  └─────────────────────────┘

  功率输出侧（每路 3 个螺丝端子）：
  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐
  │NO COM NC│ │NO COM NC│ │NO COM NC│ │NO COM NC│
  │ 第1路   │ │ 第2路   │ │ 第3路   │ │ 第4路   │
  └───────┘ └───────┘ └───────┘ └───────┘

  NO = Normally Open（常开）= 默认断开，通电后接通
  COM = Common（公共端）= 接电源进来
  NC = Normally Closed（常闭）= 默认接通，通电后断开
  我们用 COM + NO：默认断开，Pi 发信号时才通电
```

**操作：**

用**母对母**杜邦线，从 Pi 的排针接到继电器模块的排针：

| 从 Pi | 到继电器模块 | 线的颜色建议 |
|-------|-----------|------------|
| 物理针脚 2 或 4（5V） | VCC | 红色 |
| 物理针脚 6（GND） | GND | 黑色 |
| 物理针脚 12（GPIO18） | IN1 | 黄色 |
| 物理针脚 16（GPIO23） | IN2 | 绿色 |
| 物理针脚 18（GPIO24） | IN3 | 蓝色 |

同时，从继电器模块的 **GND** 再引一根线到 **面包板 GND 总线**。

> **等等，为什么继电器用 Pi 的 5V 而不是降压模块的 5V？**
>
> 都可以。Pi 的 5V 针脚其实就是直通 USB-C 供电的，所以本质上还是降压模块的 5V。从 Pi 接线更方便（距离近，用杜邦线直连）。如果你觉得 Pi 上引出 5V 不保险，也可以直接从降压模块 OUT+ 单独引一根线到继电器 VCC。

> **跳线帽设置：** 你的继电器模块上可能有 S1-S4 的跳线帽，用来选择高电平触发还是低电平触发。**先不管它，默认就行**，后面软件里可以配置。到时候测试的时候再确认：GPIO 输出 HIGH 时继电器应该吸合（听到"咔"一声），如果反了，改跳线帽或改软件配置都行。

---

### 第七步：继电器模块接线 — 功率侧（12V 气泵 + 阀门）

**做什么：** 把 12V 电源通过继电器连到气泵和两个电磁阀。

**为什么：** 继电器就像一个遥控开关。COM 端接电源（12V），NO 端接设备。Pi 发信号 → 继电器吸合 → COM 和 NO 接通 → 设备通电。

**操作：**

先接 12V 电源到三个继电器的 COM 端：

1. 从 12V 电源适配器的 **+（正极）**（就是接在降压模块 IN+ 的那根线）引出 3 根线（可以在面包板上分出来，或用接线端子分线）
2. 分别接到：
   - 第 1 路继电器 **COM1** → 12V+
   - 第 2 路继电器 **COM2** → 12V+
   - 第 3 路继电器 **COM3** → 12V+

再接设备：

3. 第 1 路继电器 **NO1** → **气泵的正极线**（通常是红色）
4. **气泵的负极线**（通常是黑色）→ **面包板 GND 总线**

5. 第 2 路继电器 **NO2** → **左腿电磁阀的正极线**（通常是红色）
6. **左腿电磁阀的负极线**（通常是黑色）→ **面包板 GND 总线**

7. 第 3 路继电器 **NO3** → **右腿电磁阀的正极线**（通常是红色）
8. **右腿电磁阀的负极线**（通常是黑色）→ **面包板 GND 总线**

```
电路回路示意（以气泵为例）：

  12V+  ──→  COM1  ···继电器开关···  NO1  ──→  气泵+
                                              气泵-  ──→  GND 总线  ──→  12V-

  Pi GPIO18 给 IN1 高电平
    → 继电器吸合
    → COM1 和 NO1 接通
    → 12V 通过气泵形成回路
    → 气泵转
```

> **怎么分辨正负极？** 气泵和电磁阀一般有两根线：红色 = 正极，黑色 = 负极。如果都是同色线，看设备上有没有标 `+` `-`。如果真的分不清，**短暂**接反不会烧（直流电机/电磁阀方向反了只是不工作或反转），试一下就知道了。

---

### 第八步：MPR121 触摸传感器接线

**做什么：** 把触摸传感器通过 I2C 总线连到 Pi。

**为什么：** MPR121 通过 I2C 通信协议告诉 Pi 哪些触摸通道被触发了。I2C 只需要两根信号线（SDA 数据 + SCL 时钟）加上供电。

**认识 MPR121 模块的接口：**

```
MPR121 模块的针脚（通常一侧是排针）：

  VCC  — 供电正极（接 3.3V，不是 5V！）
  GND  — 接地
  SDA  — I2C 数据线
  SCL  — I2C 时钟线
  IRQ  — 中断（可以不接）
  ADDR — 地址选择（可以不接，默认 0x5A）
```

**操作：**

用**母对母**杜邦线：

| 从 Pi | 到 MPR121 | 说明 |
|-------|----------|------|
| 物理针脚 17（3.3V） | VCC | 供电。**必须接 3.3V，不能接 5V！** MPR121 是 3.3V 器件，接 5V 会烧 |
| 物理针脚 6 或 9 或 14 等任意 GND | GND | 接地 |
| 物理针脚 3（GPIO2 / SDA） | SDA | I2C 数据 |
| 物理针脚 5（GPIO3 / SCL） | SCL | I2C 时钟 |

MPR121 的 GND 也接到**面包板 GND 总线**。

> **IRQ 和 ADDR 针脚不用接。** IRQ 是可选的中断通知，我们用轮询方式读取。ADDR 不接默认地址是 0x5A，软件里已经配好了。

**触摸电极怎么接（后面做）：**
MPR121 的另一侧有 0-11 共 12 个电极焊盘。到时候人员 A 会用导线或铜箔贴片从这些焊盘引出，贴到 Blob 身体各个部位。现在不用接，先把 MPR121 和 Pi 之间的 4 根线接好就行。

---

### 第九步：气管连接

**做什么：** 用硅胶气管把气泵、电磁阀和硅胶腿连起来。

**为什么：** 气泵产生气压，通过气管送到阀门，阀门控制气体进入哪条腿。

**气路拓扑：**

```
                     6mm 硅胶管
                    ┌──────────→ 电磁阀1 入口 → 电磁阀1 出口 ──→ 左腿气腔
气泵出气口 ──→ T 型三通 ┤
                    └──────────→ 电磁阀2 入口 → 电磁阀2 出口 ──→ 右腿气腔
```

**操作：**

1. **剪三段气管：**
   - 气泵出气口 → T 型三通：约 10-15cm
   - 三通 → 电磁阀 1：约 10-15cm
   - 三通 → 电磁阀 2：约 10-15cm
   - 电磁阀 1 → 左腿：根据实际距离
   - 电磁阀 2 → 右腿：根据实际距离

2. **接气泵：** 气泵有一个出气口（标 OUT 或有箭头标识），把气管套上去。如果松，用扎带或热缩管固定。

3. **T 型三通：** 把气泵出来的管子接到三通的一端，另外两端分别接两个阀门的入口。

4. **电磁阀方向：** 电磁阀有两个口，一般标有箭头表示气流方向。气体从入口进、出口出。如果没标，试一下就知道——通电后从一头吹气，能吹通的那个方向就对了。

5. **接硅胶腿：** 电磁阀出口接气管，气管另一头插进硅胶腿的气嘴。

> **没有 T 型三通怎么办？** 买一个「6mm 气管三通接头」（几毛钱）。或者临时方案：把气泵出来的管子用剪刀剪个侧孔，插入另一根管子，用热熔胶密封。丑但能用。

## 上电测试

**一切接好，检查表全打勾之后，才插电。**

### 测试 1：电源

1. 插上 12V 电源适配器
2. 用万用表测降压模块 OUT+ 和 OUT- 之间的电压 → 应该是 **4.9-5.2V**
3. 如果电压不对，**立即拔掉电源**，检查接线

### 测试 2：Pi 启动

1. 确认降压模块输出正确后，把 USB-C 线插入 Pi
2. Pi 的红色 LED 亮起 = 供电正常
3. Pi 的绿色 LED 闪烁 = 系统启动中
4. 等待启动完成（约 30 秒）

### 测试 3：圆屏

1. Pi 启动后，圆屏应该亮起来显示桌面画面
2. 如果不亮：检查 HDMI 线是否插紧、驱动板 USB 供电是否接好

### 测试 4：继电器

1. SSH 登录 Pi（或接键盘鼠标），打开终端
2. 执行以下命令测试 GPIO18（气泵继电器）：
   ```bash
   # 安装 GPIO 工具（如果没有的话）
   sudo apt-get install -y python3-gpiozero

   # 测试：让 GPIO18 输出高电平
   python3 -c "
   from gpiozero import LED
   import time
   relay = LED(18)
   relay.on()     # 应该听到继电器 '咔' 一声
   time.sleep(2)
   relay.off()    # 又 '咔' 一声，松开
   "
   ```
3. 如果听到继电器咔嗒声 = 接线正确
4. 如果没反应 = 检查 VCC/GND 是否接好，或者试试把跳线帽换到另一个触发模式
5. 同样测试 GPIO23 和 GPIO24

### 测试 5：触摸传感器

1. 在 Pi 终端执行：
   ```bash
   sudo i2cdetect -y 1
   ```
2. 应该在地址 **0x5a** 处看到一个数字，表示 MPR121 被正确检测到
3. 如果显示全是 `--`：检查 SDA/SCL 是否接对、VCC 是否接了 3.3V

### 测试 6：气泵和阀门

1. 确认气管都接好
2. 用上面测试 4 的方法，先开一个阀门（GPIO23 HIGH），再开气泵（GPIO18 HIGH）
3. 你应该能听到气泵响，并且对应的硅胶腿开始充气
4. **不要让气泵连续运行超过 30 秒！** 测试完立即关闭

## 安全注意事项

1. **接线时必须断电。** 所有线接好再通电，不要边通电边接线
2. **MPR121 只接 3.3V，不能接 5V**，否则会烧
3. **降压模块先调压再接 Pi**，超过 5.25V 会损坏 Pi
4. **气泵不能连续运行超过 30 秒**，需要冷却。软件里有安全计时器
5. **启动气泵前先开一个阀门泄压**，这个气泵不能带压启动
6. **面包板上 GND 线不要和 12V/5V 线插在同一行！** 否则短路
7. 如果闻到烧焦味或看到冒烟，**立即拔掉 12V 电源适配器**

## 常见问题

**Q: 面包板上的线好多，乱了怎么办？**
A: 用不同颜色的杜邦线区分功能。建议：红 = 电源正极、黑 = GND、黄/绿/蓝 = GPIO 信号、白 = I2C。

**Q: 继电器不响怎么办？**
A: 三个可能：(1) VCC/GND 没接好；(2) 高低电平触发方向反了，换跳线帽；(3) GPIO 编号搞错了，重新数物理针脚号。

**Q: 气泵响了但腿不充气？**
A: 检查气管有没有漏气（管子没插紧、三通处有缝隙）。把管口堵住听有没有嘶嘶声。如果气泵响但完全没气出来，可能进出气口接反了。

**Q: Pi 启动了但圆屏不亮？**
A: (1) 检查 HDMI 线两头是否插紧；(2) 确认用的是 Pi 的 HDMI0 口（靠近 USB-C 那个）；(3) 检查驱动板 USB 供电是否接好。

**Q: `i2cdetect` 检测不到 MPR121？**
A: (1) SDA 和 SCL 是否接反了？(2) VCC 是否接了 3.3V？(3) Pi 的 I2C 是否已启用？执行 `sudo raspi-config` → Interface Options → I2C → Enable。
