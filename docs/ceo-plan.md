# Alive Blob - CEO Plan

黑客松项目评审与执行计划。2026-04-07。

## 一句话定义

一个会走路、能感知触摸、用 AIGC 生成任意人脸表情回应你的软体机器人。上传老板照片，打它一下，它哭着跑了。

## 为什么做这个产品

人需要一个能用身体触碰、会用身体回应、还带着一张有意义的脸的东西，来安放那些无处安放的情绪。

现有的情绪出口都有缺陷：
- **忍着** — 伤自己
- **发朋友圈** — 是表演
- **找人倾诉** — 需要对方有空
- **解压球/捏捏乐** — 有触感没反馈，情绪出去了但没被"接住"
- **数字解压游戏** — 有人格没触感，大脑知道是假的
- **发泄锤/沙袋** — 有力度没人格，打的是物体不是"谁"

Blob 填补的空白：**有人格 + 有触感 + 会反应。** 你打它，手感是软的（触感），打的是老板的脸（人格），它哭着跑了（反应）。

## 为什么是软体

硬的东西是机器。软的东西是生物。人类大脑 0.1 秒完成这个分类。

软体独有的四个�����力：
1. **会"呼吸"** — 气动充放气让硅胶腿肉眼可见地膨胀收缩，像肌肉
2. **会"晃"** — 硅胶弹性让每一步都不完全一样，不精确 = 有生命感
3. **你敢打它** — 硅胶打不坏，触感舒适，"打"这个动作安全、有触感
4. **看起来脆弱** — 软 = 脆弱 = 触发共情。打完会愧疚，愧疚是产品体验的高潮

## 关键工程规格

### 情绪状态机转换表

```
初始状态：Calm

触发条件                              → 目标情绪    衰减回 Calm
──────────────────────────────────────────────────────────────
单通道触发 <500ms                      → Happy       15秒无交互
触发通道与上次不在同一区域               → Curious     15秒无交互
同一通道保持 >2秒                      → Shy         20秒无交互
1秒内 >=3 次触发                      → Excited     10秒无交互
>=3 通道同时触发                       → Grumpy      15秒无交互
>60秒 无任何触摸                       → Sleepy      任意触摸回 Calm

Grumpy 特殊行为：
  Grumpy 状态下继续被拍打 → 步态加速到极限（step_ms 从 600→200）
  视觉上表现为"一瘸一拐疯狂蹬腿"，即 Demo 中的"逃跑"效果
  不需要独立的 Fleeing 状态，复用 Grumpy 的 intensity 参数实现
  对应视频仍为 grumpy.mp4（哭脸），不需要额外视频文件

所有非 Calm 情绪在指定时间无交互后线性衰减回 Calm。
所有循环音效（loop）在情绪退出时立即停止（on_exit stop）。
手机端 set_emotion 指令可强制覆盖状态机。
```

### 角色包资产规格

```
目录结构：characters/<name>/
文件命名：<emotion>.mp4（calm.mp4, happy.mp4, ...共 7 个）
分辨率：  480x480（方形渲染，圆形裁切由 Eye App CSS border-radius: 50% 实现）
编码：    H.264 Baseline Profile
帧率：    30fps
时长：    3-5 秒，首尾帧视觉连贯（最后 1 帧与第 1 帧表情/姿态一致，可用剪辑软件复制首帧到末尾）
音频：    无（视频 muted，音效由独立音频文件提供）
元数据：  manifest.json 由服务器自动生成，示例：
          {"name":"boss","videos":{"calm":"calm.mp4","happy":"happy.mp4",...},"sounds":{"grumpy":"sounds/grumpy.mp3",...}}

音效文件：characters/<name>/sounds/
文件命名：<emotion>.mp3
时长：    1-3 秒，可循环
```

### 声音系统

Spec 里视频设置了 `muted`。一个"活的"东西不发声，说服力减半。**必须新增。**

实现方式：Eye App（Pi 上全屏 Chromium 打开的网页，运行视频播放器+触摸特效层）里加 `<audio>` 元素。

| 情绪 | 音效 | 触发方式 | 说明 |
|------|------|---------|------|
| Grumpy | 呜呜哭声 | on_enter + loop | 进入情绪时开始循环播放 |
| Happy | 嘿嘿笑声 | on_enter | 进入时播放一次 |
| Sleepy | 打哈欠 | on_enter | 进入时播放一次 |
| Shy | 小声嘤嘤 | on_enter | 进入时播放一次 |
| Excited | 哇哇叫 | on_enter + loop | 进入时开始循环播放 |
| Curious | 嗯？ | on_enter | 进入时播放一次 |
| Calm | 静音 | — | 无音效 |

额外：任何 MPR121 电极激活事件均触发一个短促"啪"音效（touch_feedback），不受情绪状态影响。

来源：AIGC 生成（如 ElevenLabs SFX）或 freesound.org。工作量约 30 分钟。

### Demo 模式

启动 Engine 时加 `--demo` 参数，或通过极简 Web 控制页面切换。

行为：Engine 主循环不变，但每个输出通道独立容错：
- **触摸传感器失败** → Engine 跳过触摸读取，情绪仅由 Web 控制页面驱动
- **气动系统失败** → Engine 跳过 GPIO 输出，仅发送视频切换消息
- **圆屏/视频失败** → Engine 照常驱动步态，体验降级为"触摸响应走路机器人"

每个输出通道在 `config.py` 中有独立的 `ENABLE_TOUCH / ENABLE_GAIT / ENABLE_DISPLAY` 开关。

启动方式优先级：`--demo` 启动参数（主）> Web 控制页面按钮（辅）> config.py 直接修改。

### 极简 Web 控制页面

一个 HTML 页面，包含：
- 7 个情绪按钮（Calm / Happy / Curious / Shy / Excited / Grumpy / Sleepy），点击发送 `POST /api/set_emotion`
- Demo 模式开关（toggle ENABLE_TOUCH / ENABLE_GAIT / ENABLE_DISPLAY）
- 当前情绪状态显示（通过 WebSocket state_sync 更新）

无其他功能。无 CSS 框架。原生 HTML + 几十行 JS。
